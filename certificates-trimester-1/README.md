## 📚 About this Course

> **IBM Open Source Foundations** — delivered via [IBM SkillsBuild](https://skillsbuild.org)

This course covers the core principles of open source software, IBM's involvement in the open source ecosystem, and practical skills for contributing to open source projects.

---

## 🧠 What I Learned

### 🔓 Open Source Fundamentals
- What open source software truly means (beyond just "free")
- The **Open Source Way**: Transparency · Collaboration · Deliver Early & Often · Inclusion · Community
- Common misconceptions about open source licensing

### 🏢 IBM & Open Source
- IBM's **6 open source elements**: Tooling · Consuming · Contributing · Recognition · Training · Execution
- IBM's key contributions: **Linux Kernel**, **Node-RED**, **Hyperledger**, **Kubernetes**
- How IBM leverages open source at an enterprise level

### 🤝 Contributing to Open Source
- How to find projects via GitHub's **"good-first-issue"** label
- Types of contributions: code, documentation, advocacy, community
- Repository anatomy: `README` · `LICENSE` · `CONTRIBUTING` · `CODE_OF_CONDUCT`
- The difference between **advocacy** (blog posts, talks) and **contribution** (code, docs, translation)

### 🛠️ Git & Version Control
- Git as a **distributed version control system**
- Key commands: `git status`, `git log`, `git clone`, `git commit`
- Concepts: **fork**, **branch**, **pull request**, **CI/CD**
- Platforms: **GitHub** & **GitLab** as Git hosting services

### ⚖️ Licensing & Governance
- Difference between **Copyleft** (GPL) and **Permissive** (MIT, Apache) licenses
- Why you never create your own license
- **BDFL** (Benevolent Dictator for Life) governance model
- What makes a vibrant open source community: responsive leadership · transparency · positive experience
